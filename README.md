# DataSense

Chat interface for asking questions about Indian government open datasets. Answers are grounded in live rows from [data.gov.in](https://data.gov.in) and generated by [Groq](https://groq.com) (`openai/gpt-oss-120b`).

## Architecture

```
Browser (Next.js UI)
    → POST /api/chat  { message, datasetId, history }
        → lib/data-gov-in/client   fetch dataset rows (API key server-side only)
        → lib/ai/chat-service      build system prompt + call Groq stream
    ← text/plain stream (markdown answer)
```

| Path | Role |
|------|------|
| `src/lib/env.ts` | Validates server environment variables |
| `src/lib/datasets/registry.ts` | Maps UI dataset keys → data.gov.in resource IDs |
| `src/lib/data-gov-in/client.ts` | Fetches JSON records (primary + fallback URL) |
| `src/lib/groq/client.ts` | Streams chat completions from Groq |
| `src/lib/ai/chat-service.ts` | Orchestrates data load + LLM call |
| `src/app/api/chat/route.ts` | HTTP API used by the chat UI |
| `src/hooks/useChat.ts` | Client hook: streaming fetch + message state |

API keys **must** stay in `.env.local` on the server. They are never sent to the browser.

## Setup

1. **Install dependencies**

   ```bash
   cd datasense
   npm install
   ```

2. **Configure environment**

   ```bash
   cp .env.example .env.local
   ```

   Edit `.env.local`:

   - `DATA_GOV_IN_API_KEY` — from [data.gov.in](https://data.gov.in) (register → API key)
   - `GROQ_API_KEY` — from [Groq Console](https://console.groq.com)

3. **Run the app**

   ```bash
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000).

4. **Use the app**

   - Pick a dataset from the profile menu (bottom-left).
   - Ask a question about that data (trends, comparisons, filters, summaries).
   - The server loads a sample of rows, injects them into the model context, and streams the answer.

## Datasets (Census 2011)

All default datasets are **2011 Census** tables from data.gov.in:

| Sidebar key | Dataset |
|-------------|---------|
| `sales` | Primary Census Abstract 2011 — India |
| `crm` | Population by age, residence & sex, 2011 |
| `ops` | Literacy & workers (PCA 2011, focused prompt) |

Resource IDs live in `src/lib/datasets/registry.ts`. Override in `.env.local`:

```env
DATA_GOV_RESOURCE_SALES=0764657f-00ec-4c6b-9ece-2d7b8a7401fa
```

To discover IDs for other catalogs, run `node scripts/find-census-resource.mjs` (requires `DATA_GOV_IN_API_KEY` in the environment).

## Security

- Do not commit `.env.local`.
- If an API key was shared in chat or email, **rotate it** in the provider console.
- Production: set the same variables in your host’s secret manager (Vercel, etc.).
